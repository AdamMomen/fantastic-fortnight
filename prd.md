# 📄 PRD — Approval Clarity Layer

---

## 1. Overview

**Approval Clarity Layer** is a lightweight application that transforms raw approval requests into clear, human-understandable decisions.

It helps users answer instantly:

- What will happen?
- What changes?
- How risky is this?
- Should I approve it?

This project is a **spec-driven prototype**, inspired by how approvals work in :contentReference[oaicite:0]{index=0}.

It does **not require access to Town systems** and instead focuses on improving the **decision-making layer** around approvals.

---

## 2. Problem

Users are often asked to approve actions generated by systems (AI, workflows, automation).

These approvals are frequently:

- unclear
- technical
- missing context
- risky without explanation

This leads to:

- hesitation and delay  
- incorrect approvals  
- blind trust or complete distrust  
- poor user experience  

In systems like Town, where workflows can:
- send emails
- update financial data
- schedule meetings
- submit documents  

👉 **unclear approvals create real risk**

---

## 7. Core Concept

The system takes an approval object and transforms it into:

- explanation  
- risk classification  
- before/after impact  
- recommendation  

👉 Turning **system actions → human decisions**

---

## 8. Core Features

---

### 8.1 Approval Inbox

Displays list of pending approvals.

Each row shows:
- title
- risk level
- timestamp

Goal:
👉 fast scanning and prioritization

---

### 8.2 Approval Detail Drawer (Core Feature)

Displays full breakdown:

#### Sections:

- What will happen
- Risk level
- Data used (optional)
- Before / After comparison
- Recommendation

Goal:
👉 eliminate ambiguity

---

### 8.3 Risk Indicator

- Low → Green  
- Medium → Yellow  
- High → Red  

Goal:
👉 instant visual understanding

---

### 8.4 Before / After Diff

Shows:
- current state
- resulting state

Goal:
👉 make impact visible

---

### 8.5 Recommendation Layer

Provides guidance:
- “Safe to approve”
- “Review before approving”
- “Potentially risky”

Optional:
- confidence score

Goal:
👉 reduce decision friction

---

### 8.6 Approve / Reject Actions

- simple action buttons
- no real execution (MVP)

Goal:
👉 complete decision loop

---

## 9. User Flow

```md
1. User sees approval list
2. User selects approval
3. Drawer opens
4. User reads:
   - explanation
   - risk
   - impact
   - recommendation
5. User decides:
   - approve or reject
````
